# Experience and UI

## Representation choice

The strongest player-facing form for The Pit is a hybrid 2D chat-plus-diagram interface.

This is not a downgrade from 3D. It is the more native form for a game about process, recurrence, commitment, representation, and self-reference.

## Why this form fits

The game is not fundamentally about embodied traversal.

It is about:

- lawful mystery
- finite attention
- visible process state
- maps and notes as playable objects
- repeated return in altered form
- commitment and abandonment

Those are easier to stage in symbolic, graph, map, and journal interfaces than in conventional 3D exploration.

## Recommended screen grammar

### Left panel

Use for:

- chat and system log
- journal entries
- witness testimony
- oracle dialogue
- room descriptions
- replay excerpts

### Center panel

Use for:

- chamber boards
- path graphs
- semantic maps
- garden surfaces
- oracle cards
- mirror overlays
- engine schematics

### Right panel

Use for:

- active threads
- commitments
- contradictions
- debts
- statuses
- discovered clues
- thread urgency

### Bottom action row

Use for costly verbs such as:

- inspect
- compare
- annotate
- commit
- withdraw
- bind
- wait
- revisit
- project
- graft
- seal
- reconcile

## Map stance

The map should be semantic, not only geometric.

Track states like:

- settled
- blooming
- echoing
- sealed
- contradictory
- dormant
- hungry
- witnessed

This lets progression feel alive without becoming a checklist.

## Notes and traces

Player notes, inscriptions, diagrams, oracle rulings, and replay traces should become manipulable objects.

This is central, not ornamental. Representation itself is part of play in The Pit.

## Commitment verbs

The interface must make continuation and abandonment meaningful.

Useful verbs include:

- continue
- defer
- seal
- withdraw
- bookmark
- return later

If the player can safely poke everything forever, the game's core tension dissolves.

## Style rule

Prefer legible symbolic surfaces over spectacle.

The UI should feel like an instrument for thought, not a HUD pasted on top of a separate game.
